MAR

DRF / REST API

First exposure to APIs

As with the previous module, I had never been exposed to building web APIs before and absolutely loved learning, struggling and playing with API specifications and Database Schema.

Building on the learnings from previous months, the DRF module involved creating our own crowdfunding website and building an API to store and populate our website with users data. Cool things I learnt along the way: how to create a MVP, how to plan design with Figma wireframes, how to build a Database Schema (like here with LucidChart) and how to create an API specification table, handling relationship models, etc.

At the end of this module, the skeletton of the "PointJar" crowdfunding platform was created. The idea of PointJar is to help non-for-profits finance specific local projects using people's donations of their loyalty points, miles or cash.

👉 Still work in progress, meet PointJar!

May

Group Project

We-just-killed-it!

I wish this group project never had to end... My amazing team and I worked relentlessly at creating a solution that solved our client's problem.

Our client runs in-person workshops all the time and uses a physical board to add students' sticky notes describing their "wins". One of the main problems was that the client didn't know how to make these "win walls" online and COVID-friendly.

Our solution was to take the wall of sticky note concept and digitize it. And Win{Wall} was born: an online collaboration tool that provides event organisers and their attendees an online workspace where they can add sticky notes to digital walls and share wins with others during live events

This fun project helped us reinforce all what we learnt during the She Codes programmes and got us better at designing wireframes using Figma (see our initial design concept here, using Github as a team, and deploying websites and API in Heroku.

👉 Meet the 'Win{Wall}' app!
